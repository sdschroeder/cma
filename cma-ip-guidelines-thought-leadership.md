## Content Type Guidelines

### **Blog Posts & Articles**

**Share:** Problem analysis, scientific backing, high-level concepts, business outcomes
**Protect:** Implementation details, technical specifications, proprietary methodologies
**Length:** Focus on one Green Zone topic per post for maximum impact

### **Whitepapers**

**Share:** Comprehensive problem definition, research validation, framework overview, business case; Four Pillars and Ultra-Intelligent Content; Strategic Asset Score component names and weights and analytics feedback (as in official CMA whitepaper); StoryCycle Genie reference architecture at whitepaper level (phases, high-level agent roles, outcomes)
**Protect:** Implementation blueprints, detailed technical architecture, specific algorithms, API specs, semantic schemas, handoff protocols
**Structure:** 60% problem/validation, 40% high-level solution with CTA for details. The official [cma-whitepaper.md](cma-whitepaper.md) is the compliance benchmark for what whitepapers may share.

### **Speaking Engagements**

**Share:** Framework concepts, competitive differentiation, transformation vision, audience Q&A
**Protect:** Technical implementation, client-specific details, proprietary processes
**Approach:** Position as strategic visionary, not technical implementer

### **Case Studies**

**Share:** Business value proposition, transformation potential, industry applications, strategic vision
**Protect:** Implementation methodologies, technical details, configuration specifics
**Focus:** Value proposition and competitive advantage potential, not specific implementation approaches
